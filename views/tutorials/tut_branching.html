Branching

General Knowledge

Why should you need to know this?

Normally, a computer will do everything you tell it to, one by one.

Let's say you don't want this. For an example, if you're looking for flowers to give someone.

They might be allergic to a certain kind of flower, so you'd obviously want to avoid picking those.

That's what branching is for. It lets you decide what actions to take depending on what you're doing.

What actually is this?

Branching is the most basic way to begin more complicated programming. For this tutorial, we'll focus on if-else blocks. There are three options for you.

"If" is followed by a equation, like the variable 'X' being greater than some number.

"Else" is not followed by a equation, because it holds everything that happens if the "if" part is false.

"Else if" is a special option, in case you want to set up multiple checks using the same block.

How are you supposed to set this up and use it?

The block system has a if-else block set up and ready to go.

Just drag is onto the working field, and start making your branches.

The little cog symbol in the top left of the block opens a small menu, which lets you add more if you want to. Not sure why you would? Keep on reading!

		Two Branches

		
		
		

		Why should you need to know this?

		

		This lets you begin moving into more complicated stuff.


		If you're walking around and like the look of some flowers, you might want to take some home.


		Well, the white flowers are a little boring looking, while the red flowers are great.


		So you'd use two branches to say: "When the flower is white, ignore it. When the flower is red, pick it up!"
		

		
		

		What actually is this?

		

		This can actually take two different forms, and really depends on what you want to do.


		The first is still very basic, the "if-else".


		All this does is say "If this statement is true, do this thing. If it isn't true, do this other thing." Simple.


		The other, though, is where things get a little more complicated.


		The "If-elseif", where you set up two different statements.


		This says "If this equation is true, do thing one, if it isn't, check equation two. If equation two is true, do thing two. If neither are true, move on."


		As you can see, it's a little more complicated, but still very understandable.
		

		
		

		How are you supposed to set this up and use it?

		

		First, choose which type you want. "If-else", or "If-elseif".


		Either way, you'll want to click on the little cogwheel in the top left of the "if" block.


		Then, drag either the "else" or "else-if" blocks onto the main "if" block, just like other blocks.


		Then, fill in the blocks with whatever things you want to run.
		

		
	

		
	
	

		

		Three(Or more) Branches

		
		
		

		Why should you need to know this?

		

		This lets you set up the most complicated and interesting options.


		While you're walking around looking for flowers, you'll see a variety of colors.


		If you want to only ever pick up certain colors, you'd use this to do that


		

		
		

		What actually is this?

		

		The most complex form of branching, technically going as high as you care to set up.


		It combines all three possible options. "If", "Else-If", and "Else".


		This means you can check the first equation, then the second one, and if neither are true, do something else.


		It's the natural combination of everything up to this point.


		Obviously, if you don't want to have the "Else" statement, it's not a requirement.


		Just run all "If" and "Else-If" statements.
		

		
		

		How are you supposed to set this up and use it?

		

		Once again, we go to our trusty little corner cog.


		Add as many "else-if" blocks as you like, and add the "else" block if you want to.


		

		
	

		
	
	

		

		Multiple Actions in each branch?

		
		
		

		Why should you need to know this?

		

		Well, in case you want to do multiple things in any branch of course!


		Your program won't actually know what kind of flower you're trying to pick up, until you tell it to check.


		So when you're walking around and find a flower, you want to have a block to check what color it is before picking it up.


		

		
		

		What actually is this?

		

		Instead of having a single instruction for each branch, try fitting in more than one.


		This is how you get the most out of each branch.


		Doing multiple things might mean some branches get larger than others, but that just means they can hold heavier fruit!


		

		
		

		How are you supposed to set this up and use it?

		

		Luckily for you, the blocks handle this perfectly!


		If you just drag multiple blocks into the right area, they should all slot right in!
