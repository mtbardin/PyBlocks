Branching

General Knowledge

Why should you need to know this?

Normally, a computer will run all the commands you tell it to, one by one.

Branching lets you tell it to run certain ones, depending on the condition you set up. This allows you to set up your own system of checks that cause the program to "branch" out, similar to a tree.

It's one of the core parts of programming, and something you'll be using pretty consistently.

What actually is this?

Branching is the most basic way to begin more complicated programming. For this tutorial, we'll focus on if-else conditions. There are three options for you.

"If" is followed by a condition, like the variable 'X' being greater than some number.

"Else" is not followed by a condition, because it holds everything that happens if the "if" statement is false.

"Else if" is a special option, in case you want to set up multiple checks using the same statement.

How are you supposed to set this up and use it?

The block system has a if-else block set up and ready to go.

Just drag is onto the working field, and start making your branches.

The little cog symbol in the top left of the block opens a small menu, which lets you add more if you want to. Not sure why you would? Keep on reading!

		Single Branch

		
		

		Why should you need to know this?

		

		In case you every want to do something unobtrusive, of course!


		Let's say you want to tell someone to pick up yellow flowers as they walk around.


		This is how you would do that. You'll learn how to make them walk around later, but once they are...

		"IF flower = yellow"

		"    pickUp(flower)"
		

		
		

		What actually is this?

		

		The absolute most basic form of branching. A very simple "if" statement. 


		No "elses", no "else-if's", nothing to complicate it.


		Just a simple "if this statement is true, do this thing, and then move on."

		

		
		

		How are you supposed to set this up and use it?

		

		Just use the block you already know, and keep the "else" part empty.


		It really is just that simple.

		

	

		
	
	

		

		Two Branches

		
		

		Why should you need to know this?

		

		This lets you begin moving into even more complicated things.


		Let's use an example.

		If you're controlling a robot on a hike, you probably want it to avoid bears. So when presented with two different paths, would you take the path that says "Danger: Bears!", or the one that says nothing at all?


		Well, with branching you can tell your robot to avoid the dangerous bears.

		If you want to that is. Maybe you want your robot to fight bears. Who am I to judge?
		

		
		

		What actually is this?

		

		This can actually take two different forms, and really depends on what you want to do.


		The first is still very basic, the "if-else". This is what the "if" block starts with by default.


		All this does is say "If this statement is true, do this thing. If it isn't true, do this other thing." Simple."


		The other, though, is where things get a little more complicated.


		The "If-elseif", where you set up two different statements. The normal "else" is blank, but you add the "else if"


		This says "If this statement is true, do thing one, if it isn't, check statement two. If statement two is true, do thing two. If neither are true, move on."


		

		
		

		How are you supposed to set this up and use it?

		

		First, choose which type you want. "If-else", or "If-elseif".


		If you want the second one, you'll need to use the gear to add the "elseif" statement to the block, and leave the "else" bit blank.


		Otherwise, just leave it as is. The block defaults to a "If-else" statment, so you're in luck there.


		Then, fill in the block with whatever statements and lines you want to run.
		

	

		
	
	

		

		Three(Or more) Branches

		
		

		Why should you need to know this?

		

		This lets you set up the most complicated and interesting options within your programs.
		An example is what a lemonade stand owner would want to do each day.

		If they're out of lemonade, they should order more, right? Statment one. "If Lemon Stocks = 0, order more."


		But if they have enough, it might be too cold to sell lemonade. Not many people want to eat lemon ice, after all. That's statement two. "Else-If Weather = Cold, wait for tomorrow."


		But if they have enough lemons, and it's not too cold, they get to sell their wonderful lemonade. "Else, Sell lemonade."


		

		
		

		What actually is this?

		

		The most complex form a branching, technically going as high as you care to set up.


		It combines all three possible statements. "If", "Else-If", and "Else".


		This means you can check the first statement, then the second one, and if neither are true, run some other code.


		It's the natural combination of everything up to this point.


		Obviously, if you don't want to have the "Else" statement, it's not a requirement.

		Just run all "If" and "Else-If" statements, and make sure nothing happens by default.
		

		
		

		How are you supposed to set this up and use it?

		

		Once again, we go to our trusty little corner cog.


		Add as many "Else-If" checks as you like, and start filling them and their instructions in.
		

	

		
	
	

		

		Multiple Actions in each branch?

		
		

		Why should you need to know this?

		

		Well, in case you want to have multiple instructions in each branch of course!


		Going back to the bear example. If you for some reason want your robot to fight a bear, it would be better served by bringing out some anti-bear spray.


		So you'd have your statement look something like this: "If bearSign, choose that path, and pull out anti-bear spray"


		Now your robot is well protected as they hunt their bears.
		

		
		

		What actually is this?

		

		Instead of having a single instruction for each branch, try fitting in more than one.


		This is how you get the most out of each branch.


		Doing multiple things might mean some branches get larger than others, but that just means they can hold heavier fruit!


		

		
		

		How are you supposed to set this up and use it?

		

		Luckily for you, the blocks handle this perfectly!


		If you just drag multiple blocks into the right area, they should all slot right in!
